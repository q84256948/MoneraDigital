## 2. 产品逻辑与进度透明化 (User Trust Design)

### 2.1 申购全生命周期状态
为了消除用户在申购确认期的“信任黑洞”，UI 必须呈现清晰的进度条：
1. **已提交 (Submitted)**：T日，资产已锁定，等待系统撮合。
2. **撮合中 (Matching)**：T+1日 10:00 - 12:00，系统正在根据额度进行最终分配。
3. **已确认/计息中 (Earning)**：T+1日 12:00 后，订单正式生效。
4. **申购失败 (Failed)**：额度不足，资金实时退回个人钱包。

### 2.2 自动申购 (Auto-Reinvestment) 强化逻辑
- **机制**：到期日自动以“本金 + 累计利息”续购。
- **利率策略**：利率变产品不变。
- **【新增】失败回退机制**：
    - 若续购时产品已售罄或用户触发合规限制（如 IP 变更），系统必须在 **15 分钟内** 将资金退回个人钱包。
    - 触发“续购失败”高优先级通知（App Push + 邮件），告知失败原因并引导手动操作。

## 3. 合规与交互优化 (Anti-Error Design)

### 3.1 IP 地理围栏 - 前置校验
- **交互要求**：严禁在点击提交后才报错。
- **实时反馈**：当检测到新加坡 IP 时，若金额输入框数值 < 10,000 USDT，输入框下方立即显示红色提示语：“根据合规要求，当前地区起购金额为 10,000 USDT”，并禁用提交按钮。

### 3.2 确认与预警邮件
- **时间**：到期前 3 天。
- **UX 增强**：邮件中需对比“自动续购”与“手动提取”的预期收益差（复利展示），并提供一键跳转开关的深层链接（Deep Link）。

---

## 4. 收益透明度与利息计算

### 4.1 收益视觉化 (Yield Calendar)
- **要求**：前端需提供“收益日历”视图，展示每日利息发放记录。
- **利息公式**：`日利息 = 申购数量 * (年化利率 / 365)`。
- **复利展示**：在开启自动申购开关时，动态计算并展示“连续续购 3 期”后的预期总年化收益率。

## 5. Safeheron 安全背书设计 (Security Branding)

### 5.1 品牌心智转化
- **提币状态描述**：将原本冰冷的“审核中”文案优化为：“**资金正处于 Safeheron 硬件冷钱包多签保护中，预计 [X] 分钟内完成审批**”。
- **安全标识**：在申购和提币页面显著位置展示“Secured by Safeheron”图标，将技术上的“慢（多签）”转化为品牌上的“稳（安全）”。

---

## 6. 极端情况处理 (Edge Cases Handling)

### 6.1 汇率与滑点 (Slippage)
- **规则**：若申购涉及非 USDT 币种转换，T+1 确认时的汇率波动超过 0.5%，系统应自动撤销申购并退款，防止用户由于汇率剧烈波动遭受意外损失。

### 6.2 提前赎回阶梯费率 (Tiered Penalty)
若后台开启“允许提前赎回”，需执行阶梯违约金政策：
- **持有时间 < 50%**：扣除全部已发放利息。
- **持有时间 ≥ 50% 但 < 100%**：扣除 50% 已发放利息。
- **手续费**：固定 1% 的本金手续费。

### 6.3 接口与 Webhook 补偿机制
- **幂等性**：Safeheron 的所有 Webhook 必须支持幂等处理。
- **自动对账**：系统每 1 小时自动调用 Safeheron `get_withdraw_request_status` 接口，补齐由于网络抖动丢失的 Webhook 通知，确保用户余额更新不延迟。

## 7. 页面交互细节 (UI/UX Upgrades)
- **稀缺性引导**：在理财首页展示“火热申购中”、“仅剩 12%”或“预计 3 小时后售罄”等实时标签。
- **文案规范**：
    - “我的钱包” -> “我的资产”
    - “剩余可投” -> “本次可投”
    - “撤销申购” -> “确定要撤销该申购吗？资金将立即退回您的钱包。”

---

## 8. 安全审计与合规
- **IP 库更新**：系统必须每周更新一次 IP 地理位置库（如 MaxMind），确保 IP 识别的实时性。
- **多签权限**：所有大于 100,000 USDT 的提币申请必须触发“3/5 多签”策略，其余执行“2/3 多签”。
